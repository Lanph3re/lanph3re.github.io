Git? GitHub?

네이버 D2 Partners 행사에 참가 신청을 GitHub으로 해야해서 신청하는 김에 계정도 만들고
GitHub 사용법을 이제 익혀봐야겠다는 생각이 들었다. GitHub랑 Git이 헷갈릴 수 있는데 내가 이해한 바로는
Git: 소프트웨어의 버전을 관리하기 위해 개발된 프로그램
GitHub: Git을 사용하는 데 편리함을 주기 위한 웹 호스팅 서비스
이다.

Git 설치

따라서 Git은 프로그램이기 때문에 설치를 해야한다. https://git-scm.com/ 에서 Git을 설치할 수 있다. 설치를 하고 나면 터미널을 열어서 다음과 같이 입력하자.
git config --global user.name "유저이름"
git config --global user.mail "메일주소"
GitHub를 이용하기 위해서는 계정을 만들어야 한다. 계정을 만들고 나서 먼저 Repository를 만들어야한다. Repository는 하나의 프로젝트 파일들을 담고 있는 폴더라고 생각하면 된다. 줄여서 Repo라고도 하는데, GitHub는 이 Repo들을 관리하기 쉽도록 만들어진 서비스고 단순히 생각해서 Repo들을 저장해둘 수 있는 드라이브같은 거라고 이해하면 편할 듯 싶다.(커뮤니티 기능이 활성화 되어 있다는 점에서 단순 드라이브는 아니긴 하다.) 나는 Data-Structure-School이라는 Repo를 만들었다. 저번 학기에 들었던 자료구조 수업의 과제 코드들을 저장하고 싶었다.
만든 repo로 들어가서 Clone or download라는 버튼을 누르면 URL이 있는데 이 URL을 복사를 한다. 이 Repo를 내 컴퓨터에 다운로드를 하고 싶다. 터미널에서 다운로드 받고 싶은 폴더에 들어간 후 명령어를 입력한다.
git clone 복사한URL

GitHub에 있는 Repo가 내 컴퓨터에 받아진 걸 알 수 있다.
여기서 짚고 넘어가야할 것이 있다. GitHub에서 만든 repo는 remote repo라고 하고
git clone또는 git init으로 내 컴퓨터에 만든 repo는 local repo라고 부른다.
이제 GitHub에 올리고 싶었던 파일들을 내 컴퓨터에 있는 Repo로 옮기면 아래 사진처럼 변한다.

초록색으로 파일이름이 나오는 것은 새로운 파일이 추가됐다는 뜻이다.
그 다음에 아래 명령어를 입력하자.
git status
그럼 아래와 같은 결과가 나온다.

git은 버전관리 프로그램이라고 했다. 버전관리를 하려면 어떤 파일에 수정이 있었는 지를 확인할 수 있어야 한다. git은 당연히 파일의 수정을 알아채는 기능이 있는데, 바로 tracking이라고 한다. 우리가 폴더에 막 추가한 파일들은 이전에 git에 등록된 적이 없기 때문에 수정 여부를 확인할 수 없고, git status의 결과에서 untracked라고 알려주는 것이다. 이제 폴더에 추가한 파일들을 git에 등록을 해야한다. 이걸 Commit이라고 한다.

Commit?

Repo에 파일을 가져다 놓기만 한다고 해서 Git에도 추가가 되는 것은 아니다. Git에게 '내가 추가한 파일이 있으니 알아둬'라는 절차가 필요하다. 이걸 Commit이라 한다. 바로 방금 언급을 했지만 '추가(혹은 수정)한 파일'에 대해서 Commit을 하는 것이다. 하지만 추가한 모든 파일에 대해서 Commit을 한다면 나름대로 고충이 있을 수 있다. 따라서 Commit을 하려면 어떤 파일을 Commit할건지 Commit할 파일 목록을 등록해야한다. 목록에 파일을 추가하려면 
git add filename

과 같은 결과가 나온다. git add를 입력해서 목록에 추가된 파일은 Commit이 될 준비가 됐다는 뜻에서 Staged 상태가 된다. 그 상태에서 이제 Commit을 해보자.
git commit -m "message"
-m은 commit에 대해 어떤 commit인지를 알려주기 위해 설명을 붙이는 옵션이다.
이제 commit까지 완료함으로써 컴퓨터에 있는 local repo에만 새로운 파일들을 정식으로 추가한 것이다.
이제 GitHub에 있는 remote repo에도 파일을 추가하고 싶다. 이때는 git push 명령어를 입력하면 된다.
git push
명령어를 입력하면 GitHub에 로그인하는 창이 나타나고 로그인을 하면
